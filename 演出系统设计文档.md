# 演出系统设计文档

> 前身为"音频系统"，现升级为事件驱动的广义媒体演出层（Presentation Layer）。  
> BGM 部分不变，由原有 `AudioController` 继续管理。

---

## 一、概述与职责边界

| 系统 | 职责 | 文件 |
|------|------|------|
| **AudioController**（原有） | 持续播放 BGM，随场景平滑切换 | `audio.ts` |
| **PresentationSystem**（新增） | 事件触发的一次性媒体演出：音效 + GIF动图 + 视频片段 | `presentation/` |

**关键区别：**
- BGM = 持续背景，无需触发点，调用者只关心"当前是什么场景"
- 演出 = 一次性弹出，由游戏事件触发，有时长、有优先级、有视觉层

## 三、演出系统架构

### 核心类型

```typescript
// presentation/types.ts

type PresentationPosition = "fullscreen" | "center" | "corner";

interface AudioClip {
  src: string;
  volume?: number;       // 叠加到全局音量，0~1，默认 0.78
}

interface VisualClip {
  type: "gif" | "video";
  src: string;
  position: PresentationPosition;
  width?: number;
  height?: number;
  loop?: boolean;
}

interface PresentationConfig {
  id: string;
  audio?: AudioClip;
  visual?: VisualClip;
  durationMs?: number;   // 不填则跟随音频/视频自然时长
  priority: number;      // 数字越大越重要
  blockInput?: boolean;  // true 时屏蔽操作（开局视频用）
}
```

### 优先级参考

| 优先级 | 场景 |
|--------|------|
| `0` | 普通牌效音效（reverse / skip / +2+4） |
| `1` | 角色技能发动 |
| `5` | 游戏结束结算动画 |
| `10` | 开局动画（`blockInput: true`） |

### 演出注册表结构

所有演出配置集中在 `presentation/registry.ts`，新增演出只需在此文件加一条记录：

```typescript
const PRESENTATION_REGISTRY: Record<string, PresentationConfig> = {
  "card.reverse":           { audio: { src: "/audio/sfx/reverse_igiari.mp3" },    priority: 0 },
  "card.skip":              { audio: { src: "/audio/sfx/skip_matta.mp3" },         priority: 0 },
  "card.draw_two":          { audio: { src: "/audio/sfx/draw_stack_kurae.mp3" },   priority: 0 },
  "card.wild_draw_four":    { audio: { src: "/audio/sfx/draw_stack_kurae.mp3" },   priority: 0 },
  "skill.naruhodou.judgment": {
    audio: { src: "/audio/sfx/龙之介在这.mp3" },
    priority: 1, durationMs: 2500
  },
  "game.intro": {
    // visual: { type: "video", src: "/video/game_intro.mp4", position: "fullscreen" },
    // 视频资源就绪后取消注释
    priority: 10, blockInput: false, durationMs: 0
  },
  "game.result.win":  { priority: 5, durationMs: 0 },   // 视觉资源待补充
  "game.result.lose": { priority: 5, durationMs: 0 },
};
```

## 四、触发机制

### 客户端自行推断（牌效）

`gameStore.ts` 在 `statePatch` 里检测弃牌堆顶牌变化：

```
topCard.id 改变 → triggerPresentation("card." + topCard.kind)
```

无需服务端配合，纯前端推断。

### 服务端 hint（技能、特殊事件）

`StatePatchEvent`（通过 `GameViewEventBase`）新增可选字段：

```typescript
presentationHint?: string;   // e.g. "skill.naruhodou.judgment"
```

服务端在 skill 发动成功后，把 `skill.presentationId` 附带进广播。  
客户端收到后调用 `triggerPresentation(event.presentationHint)`。

### 游戏生命周期事件

| 事件 | 触发演出 | 在哪触发 |
|------|---------|---------|
| `gameStart` | `"game.intro"` | `gameStore.ts` |
| `gameOver`（赢） | `"game.result.win"` + `playResultBgm(true)` | `gameStore.ts` |
| `gameOver`（输） | `"game.result.lose"` + `playResultBgm(false)` | `gameStore.ts` |

## 五、队列与优先级规则

1. 当前无演出 → 直接播放
2. 新演出 > 当前优先级 → 中断当前，立即播放
3. 新演出 = 优先级 0（牌效）→ 丢弃队列中已有的优先级 0 条目，加入一条新的
4. 其他情况 → 追加到队列末尾
5. 当前演出结束（超时 or 视频 `onEnded`）→ 取队列下一条

## 六、文件结构

```
apps/client/src/
  presentation/
    types.ts              ← PresentationConfig 等类型
    registry.ts           ← PRESENTATION_REGISTRY 注册表
    store.ts              ← Zustand slice（current / queue / trigger / dismiss）
  components/
    PresentationOverlay.tsx  ← React 渲染层（固定 z-index 叠在游戏上方）

apps/client/public/
  audio/
    bgm/                  ← 现有 BGM（不动）
    sfx/                  ← 现有 SFX
  video/                  ← 新增目录（演出视频，.mp4/.webm）
  animations/             ← 新增目录（演出 GIF 动图）
```

## 七、角色技能与演出的整合

角色定义 `SkillDefinition` / `SkillPublicInfo` 新增 `presentationId?: string`。  
服务端 skill handler 在 `onActivate` 成功后，把 `skill.presentationId` 放进 `presentationHint` 广播。

**开发新角色时增加演出只需三步：**
1. 把音效/视频/GIF 放到对应 `public/` 子目录
2. 在 `presentation/registry.ts` 加一条记录
3. 在角色定义里写 `presentationId: "skill.xxx.yyy"`

## 八、已知局限与后续方向

| 问题 | 影响 | 解决方向 |
|------|------|---------|
| 视频文件暂无 | `game.intro` / 结算动画无视觉效果 | 待资源就绪后直接更新 registry |
| GIF 动图暂无 | 牌效只有音效，无视觉 | 同上 |
| 演出仅有单全局 Audio 播放节点（HTML5 Audio） | 可能与 BGM 竞争 | 演出音效用独立 Audio 实例（已在 `playSfxByName` 中实现） |
| `syncGameBgm` 优势/劣势 BGM 切换仍未接入 | 游戏中 BGM 不随局面变化 | 在 `statePatch` handler 中调用 `syncGameBgm` |
